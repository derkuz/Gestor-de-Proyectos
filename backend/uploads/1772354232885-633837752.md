# Turnero IPROSS

Este proyecto es un sistema de gestión de turnos.

## Base de Datos - Migraciones con Alembic

Las migraciones de la base de datos se gestionan con Alembic y se aplican automáticamente cuando la aplicación se inicia.

### Crear una nueva migración

Después de realizar cambios en los modelos de SQLAlchemy (en `backend/app/models/`), puedes generar una nueva revisión de migración automáticamente. Ejecuta el siguiente comando desde tu terminal:

```bash
docker-compose exec backend alembic revision --autogenerate -m "Una descripción clara de los cambios"
```

Esto creará un nuevo archivo de migración en `backend/alembic/versions/`. Es importante revisar este archivo para asegurarse de que los cambios generados son correctos.

### Aplicar Migraciones

No necesitas hacer nada. Las migraciones se aplican automáticamente cada vez que se inicia el servicio `backend` gracias al comando configurado en `docker-compose.yml`. Se ejecutarán todas las migraciones pendientes hasta la última versión.

### Deshacer Migraciones (Downgrade)

Si necesitas revertir una migración, puedes usar el comando `alembic downgrade`.

**Importante:** Revertir migraciones puede ser una operación delicada y podría resultar en pérdida de datos. Úsalo con precaución.

#### Revertir la última migración:

```bash
docker-compose exec backend alembic downgrade -1
```

#### Revertir a una revisión específica:

Primero, busca el ID de la revisión a la que quieres volver. Puedes ver el historial con:
```bash
docker-compose exec backend alembic history
```
Luego, usa el ID de la revisión en el comando:
```bash
docker-compose exec backend alembic downgrade [ID_de_revisión]
```

#### Revertir todas las migraciones (volver a una base de datos vacía):

Esto eliminará todas las tablas gestionadas por Alembic.
```bash
docker-compose exec backend alembic downgrade base
```
