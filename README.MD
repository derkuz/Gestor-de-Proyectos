## 6. Modelo de Datos (Entidades)

### 6.1 Proyectos (`Project`)
- `id`: UUID (PK)
- `nombre`: String
- `descripcion`: Text
- `estado`: Enum (`ACTIVO`, `PAUSADO`, `FINALIZADO`)
- `fechaCreacion`: DateTime

### 6.2 Tareas (`Task`)
- `id`: UUID (PK)
- `proyectoId`: UUID (FK)
- `nombre`: String
- `prioridad`: Enum (`BAJA`, `MEDIA`, `ALTA`, `CRITICA`)
- `estado`: String (ej: Pendiente, En Desarrollo)
- `esSubtarea`: Boolean
- `padreId`: UUID (FK, opcional)

### 6.3 Tickets (`Ticket`)
- `id`: UUID (PK)
- `codigo`: String (ej: TKT-001)
- `titulo`: String
- `mensaje`: Text
- `estado`: String
- `tareaRelacionadaId`: UUID (FK, opcional)
- `solicitanteId`: UUID (FK)

### 6.4 Usuarios (`User`)
- `id`: UUID (PK)
- `email`: String (Unique)
- `password`: String (Hash)
- `nombre`: String
- `rol`: Enum (`ADMIN`, `PROJECT_MANAGER`, `COLABORADOR`, `EXTERNO`)

### 6.5 DocumentaciĂłn (`Documentation`)
- `id`: UUID (PK)
- `proyectoId`: UUID (FK, 1:1)
- `contenido`: Text (Markdown/Mermaid)
- `ultimaActualizacion`: DateTime
